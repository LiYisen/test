# 阴阳线突破交易策略说明文档 (Strategy)

## 1. 策略概述

阴阳线突破策略是一种基于价格形态的趋势跟踪策略。该策略通过识别连续阳线和连续阴线形成的"阴阳集合"，在价格突破集合边界时产生交易信号。

### 1.1 核心理念
- **趋势识别**: 通过阴阳线元素的合成，识别当前市场的多空力量对比
- **突破交易**: 在价格突破关键支撑/阻力位时入场
- **动态止损**: 持仓期间根据阴阳集合动态调整止损/反手价位

### 1.2 策略特点
- 纯技术分析策略，不依赖基本面数据
- 适用于日线级别的趋势行情
- 支持多空双向交易
- 内置杠杆管理机制

### 1.3 策略参数

| 参数 | 类型 | 默认值 | 范围 | 说明 |
|------|------|--------|------|------|
| leverage | float | 3.0 | 0.1-10.0 | 账户承担的百分比风险系数 |

## 2. 阴阳线元素定义

### 2.1 K线方向判断
| 条件 | 方向 |
|------|------|
| Close > Open | 阳线 |
| Close < Open | 阴线 |
| Close = Open | 十字星（沿用上一方向） |

### 2.2 阴阳线元素
阴阳线元素是对连续同向K线的合并抽象：

**阳线元素 (Yang)**:
- 由连续阳线合并而成
- 记录该区间的最高价和最低价
- 表示多头力量集中的价格区间

**阴线元素 (Yin)**:
- 由连续阴线合并而成
- 记录该区间的最高价和最低价
- 表示空头力量集中的价格区间

### 2.3 元素层级
策略维护四个层级的阴阳线元素：

| 元素 | 含义 | 说明 |
|------|------|------|
| 阳1 (Yang1) | 最近一个阳线元素 | 当前或最近的多头区间 |
| 阴1 (Yin1) | 最近一个阴线元素 | 当前或最近的空头区间 |
| 阳2 (Yang2) | 次近一个阳线元素 | 已完成的多头区间 |
| 阴2 (Yin2) | 次近一个阴线元素 | 已完成的空头区间 |

### 2.4 元素更新规则

**方向相同时（合并）**:
```
如果当前K线方向与最近元素方向相同：
  - 更新该元素的High = max(元素.High, K线.High)
  - 更新该元素的Low = min(元素.Low, K线.Low)
```

**方向变化时（降级）**:
```
如果当前K线方向与最近元素方向不同：
  - 旧的阳1降级为阳2
  - 旧的阴1降级为阴2
  - 当前K线成为新的阳1或阴1
```

**示例**:
```
日期       K线方向   操作                    阳1        阴1        阳2        阴2
--------------------------------------------------------------------------------
Day 1      阳线      创建阳1                 新建       -          -          -
Day 2      阳线      合并到阳1               更新       -          -          -
Day 3      阴线      阳1→阳2，创建阴1        -          新建       原阳1      -
Day 4      阴线      合并到阴1               -          更新       原阳1      -
Day 5      阳线      阴1→阴2，创建阳1        新建       -          原阳1      原阴1
```

## 3. 信号价格计算（核心）

### 3.1 无持仓时的信号价格计算

**重要区分**：无持仓有两种情况，计算方式不同：

#### 3.1.1 真正的无持仓（回测开始，从未开过仓）
当回测刚开始，从未有过任何持仓时：
```
做多信号价 = max(阴1.High, 阳1.High)
做空信号价 = min(阴1.Low, 阳1.Low)
```
**只使用阴1和阳1，不使用阴2阳2**。

#### 3.1.2 有持仓历史但当前无持仓
当曾经有过持仓但已平仓，当前无持仓时：
```
如果当前K线为阳线：
  做多信号价 = max(阴1.High, 阳1.High)
  做空信号价 = min(阳1.Low, 阴2.Low)  // 使用阴2

如果当前K线为阴线：
  做多信号价 = max(阴1.High, 阳2.High)  // 使用阳2
  做空信号价 = min(阴1.Low, 阳1.Low)
```

**解释**：一旦开过仓，就进入正常逻辑，需要使用阴1阳1和阴2阳2。

### 3.2 持有多头时的做空信号价计算

持有多头仓位时，需要计算做空信号价（止损/反手价）：

| 当前K线类型 | 做空信号价计算 | 说明 |
|------------|---------------|------|
| 阳线 | min(阴1.Low, 阳1.Low) | 使用阴1阳1的低点 |
| 阴线 | min(阳1.Low, 阴2.Low) | 使用阳1和阴2的低点 |

**解释**:
- 阳线时使用阳1和阴1的低点
- 阴线时使用阳1和阴2的低点（因为阴线可能正在形成新的阴1）

### 3.3 持有空头时的做多信号价计算

持有空头仓位时，需要计算做多信号价（止损/反手价）：

| 当前K线类型 | 做多信号价计算 | 说明 |
|------------|---------------|------|
| 阳线 | max(阴1.High, 阳2.High) | 使用阴1和阳2的高点 |
| 阴线 | max(阴1.High, 阳1.High) | 使用阴1阳1的高点 |

**解释**:
- 阳线时使用阴1和阳2的高点（因为阳线可能正在形成新的阳1）
- 阴线时使用阴1和阳1的高点

### 3.4 信号价格计算示例

**场景1**: 真正的无持仓（回测开始）
```
阴阳集合状态：
  阳1: High=4000, Low=3800
  阴1: High=4100, Low=3700
  阳2: High=3900, Low=3600（忽略）
  阴2: High=4050, Low=3650（忽略）

做多信号价 = max(阴1.High, 阳1.High) = max(4100, 4000) = 4100
做空信号价 = min(阴1.Low, 阳1.Low) = min(3700, 3800) = 3700
```

**场景2**: 有持仓历史但当前无持仓，当前K线为阳线
```
阴阳集合状态：
  阳1: High=4000, Low=3800
  阴1: High=4100, Low=3700
  阳2: High=3900, Low=3600
  阴2: High=4050, Low=3650

做多信号价 = max(阴1.High, 阳1.High) = max(4100, 4000) = 4100
做空信号价 = min(阳1.Low, 阴2.Low) = min(3800, 3650) = 3650
```

**场景3**: 持有多头，当前K线为阴线
```
阴阳集合状态：
  阳1: High=4000, Low=3800
  阴1: High=4100, Low=3700
  阳2: High=3900, Low=3600
  阴2: High=4050, Low=3650

做空信号价 = min(阳1.Low, 阴2.Low) = min(3800, 3650) = 3650
```

## 4. 交易触发逻辑

### 4.1 触发判断
每根K线到达时，使用上一根K线结束后计算的信号价格进行触发判断：

```
如果 High >= 做多信号价：
  触发做多信号

如果 Low <= 做空信号价：
  触发做空信号
```

### 4.2 交易执行
**开仓**:
- 无持仓时，触发信号直接开仓
- 开仓价格 = 信号价格

**平仓反手**:
- 有持仓时，触发反向信号则平仓并反手开仓
- 平仓价格 = 信号价格
- 新开仓价格 = 信号价格

### 4.3 交易示例

**示例1: 无持仓开多**
```
日期       High   Low    做多信号价  做空信号价  操作
-------------------------------------------------------
Day 1      3950   3850   4000       3700       无
Day 2      4050   3900   4000       3700       High>=4000, 开多@4000
Day 3      4100   3950   -          3800       持有多头
```

**示例2: 多头反手做空**
```
日期       High   Low    做多信号价  做空信号价  操作
-------------------------------------------------------
Day 1      -      -      -          3800       持有多头@4000
Day 2      3950   3750   -          3800       Low<=3800, 平多@3800, 开空@3800
Day 3      3850   3700   3900       -          持有空头
```

## 5. 临时阴阳集合（重要）

### 5.1 概念说明
临时阴阳集合是在特定交易条件下生成的临时状态，**仅影响下一根K线的信号价格计算**。

### 5.2 触发条件

| 场景 | K线类型 | 是否触发 | 原因 |
|------|--------|---------|------|
| 做多开仓 | 阴线 | ✅ 触发 | 逆势开仓，需要调整信号价 |
| 做多开仓 | 阳线 | ❌ 不触发 | 顺势开仓，无需调整 |
| 做多开仓 | 十字星 | ❌ 不触发 | 方向不明确 |
| 做空开仓 | 阳线 | ✅ 触发 | 逆势开仓，需要调整信号价 |
| 做空开仓 | 阴线 | ❌ 不触发 | 顺势开仓，无需调整 |
| 做空开仓 | 十字星 | ❌ 不触发 | 方向不明确 |

### 5.3 生成逻辑

**做多 + 阴线**：
```
把阴线当成阳线处理：
  - 就近的是阴1 → 自己成为阳1
  - 就近的是阳1 → 合入阳1
```

**做空 + 阳线**：
```
把阳线当成阴线处理：
  - 就近的是阳1 → 自己成为阴1
  - 就近的是阴1 → 合入阴1
```

### 5.4 临时阴阳集合的状态方向（关键）

**重要**：临时阴阳集合有自己的状态方向（IsYang），与当前K线的实际方向不同。

| 场景 | 当前K线实际方向 | 临时集合状态方向 |
|------|----------------|-----------------|
| 做多 + 阴线 | 阴线（false） | 阳线（true） |
| 做空 + 阳线 | 阳线（true） | 阴线（false） |

**信号价格计算时**：
- 如果存在临时阴阳集合，使用临时集合的状态方向（IsYang）
- 如果不存在临时阴阳集合，使用当前K线的实际方向

### 5.5 使用规则和时序

临时阴阳集合的使用时序（**关键**）：

```
处理K线T时：
  Step 1: 如果存在临时阴阳集合（由K线T-1生成）
    → 使用临时集合计算信号价格
    → 用新计算的信号价格进行触发判断
  
  Step 2: 触发判断完成后
    → 标记临时集合为已使用
  
  Step 3: 更新阴阳集合状态
  
  Step 4: 如果成交
    → 判断是否生成新的临时阴阳集合
    → 计算反向信号价
```

**重要**：临时集合必须先用于计算信号价格，然后才能标记为已使用。

### 5.6 临时集合示例

**场景**: 做多开仓，当前K线为阴线

```
交易前状态：
  阳1: High=4000, Low=3800
  阴1: High=3900, Low=3700

交易：开多@3850（阴线，Low<=做空信号价）

临时集合生成：
  把阴线(High=3900, Low=3700)当成阳线处理
  就近的是阴1，所以自己成为阳1
  
  临时阳1: High=3900, Low=3700
  临时阴1: High=3900, Low=3700（原阴1）
  临时集合状态方向: 阳线（true）

下一根K线使用临时集合计算信号价：
  临时集合状态方向为阳线
  持有多头 + 阳线 → 做空信号价 = min(临时阴1.Low, 临时阳1.Low) = min(3700, 3700) = 3700
```

### 5.7 完整示例：4月7日-8日交易

```
4月7日：
  持仓：无
  K线：阴线（High=4535, Low=4224）
  阴阳集合：
    阳1: High=4489, Low=4236
    阴1: High=4537, Low=4280
    阳2: High=4630, Low=4440
    阴2: High=4535, Low=3950
  
  有持仓历史，当前K线为阴线：
    做多信号价 = max(阴1.High, 阳2.High) = max(4537, 4630) = 4630
    做空信号价 = min(阴1.Low, 阳1.Low) = min(4280, 4236) = 4236
  
  Low=4224 <= 做空信号价4236 → 触发做空
  但实际是做多触发（High=4535 >= 做多信号价4534）
  
  开多@4534
  K线为阴线，做多开仓 → 生成临时阴阳集合
  临时集合：把阴线当成阳线
    临时阳1: High=4535, Low=4224
    临时阴1: High=4534, Low=4224
    临时集合状态方向: 阳线（true）

4月8日：
  持仓：多头@4534
  存在临时阴阳集合
  
  Step 1: 使用临时集合计算反向信号价
    临时集合状态方向为阳线
    持有多头 + 阳线 → 做空信号价 = min(临时阴1.Low, 临时阳1.Low) = min(4224, 4224) = 4224
  
  Step 2: 触发判断
    Low=3950 <= 4224 → 触发平多开空
  
  Step 3: 标记临时集合为已使用
  
  结果：平多@4224，开空@4224
```

## 6. 杠杆计算

### 6.1 杠杆公式
```
杠杆 = (风险系数 / 100) * 开仓价格 / (开仓价格 - 止损价)
```

**参数说明**:
- 风险系数: 命令行参数 `-leverage`，表示账户承担的百分比风险
- 开仓价格: 交易成交价格
- 止损价: 根据开仓方向和阴阳集合确定

### 6.2 止损价定义

**止损价根据开仓方向确定**：

| 开仓方向 | 止损价计算 | 说明 |
|---------|-----------|------|
| 开多 | min(阴1.Low, 阳1.Low) | 使用阴1阳1的低点作为止损价 |
| 开空 | max(阴1.High, 阳1.High) | 使用阴1阳1的高点作为止损价 |

**止损价与反向信号价的区别**：
- **止损价**：在开仓时确定，用于计算杠杆，固定使用阴1阳1的高低点
- **反向信号价**：在持仓期间动态更新，用于触发止损/反手，会根据当前K线类型和阴2阳2变化

### 6.3 杠杆示例

**场景**: 
- 风险系数 = 4 (4%风险)
- 开仓价格 = 4000
- 阴1.Low = 3800, 阳1.Low = 3700
- 开多方向

```
止损价 = min(阴1.Low, 阳1.Low) = min(3800, 3700) = 3700
杠杆 = (4 / 100) * 4000 / (4000 - 3700)
     = 0.04 * 4000 / 300
     = 160 / 300
     = 0.53
```

### 6.4 杠杆限制
- 最大杠杆限制为 6.0 倍
- 当计算杠杆超过限制时，使用限制值

## 7. 移仓换月

### 7.1 主力合约识别

**初始主力合约**：
- 回测第一天，选择持仓量最大的合约作为初始主力合约

**单向切换**：
- 合约只往后续月份切换，不往回切换
- 例如当前主力是2601合约，新的主力只能是2602、2603及之后的合约，不能是2512及之前的合约

**切换条件**：
- 当后续月份合约的持仓量和成交量**同时**超过当前主力合约时，切换为新的主力合约
- 必须同时满足两个条件，缺一不可

**保持机制**：
- 如果后续月份合约只有一项指标（持仓量或成交量）超过当前主力合约，则保持当前主力合约不变
- 这避免了频繁切换，确保主力合约的稳定性

### 7.2 移仓触发
当检测到主力合约发生变化时，触发移仓换月流程。

**检测时机**: T日收盘后
**执行时机**: T+1日开盘时

### 7.3 移仓流程

```
T日收盘后：
  1. 检测主力合约是否切换
  2. 如果切换：
     a. 先处理旧主力合约当天的K线（如果有）
     b. 记录待移仓信息
     c. 更新所有合约的阴阳集合状态
     d. 记录状态历史（包含新旧合约信息）

T+1日开盘：
  1. 检查是否有待执行的移仓换月
  2. 如果有：
     a. 使用旧合约开盘价平仓
     b. 获取新合约历史K线
     c. 模拟交易确定新合约方向
     d. 使用新合约开盘价开仓
  3. 继续处理T+1日的正常交易逻辑
```

### 7.4 新合约方向确定

**模拟交易法**:
1. 获取新合约从上市首日到移仓前一日的所有历史K线
2. 使用策略模拟在新合约上连续交易
3. 以模拟结果确定的持仓方向作为新合约开仓方向

**特点**:
- 新合约方向独立于旧合约
- 方向可能与旧合约相反
- 符合策略逻辑的一致性

### 7.5 新合约信号价格

移仓换月时，新合约的信号价格根据模拟持仓状态计算：

```
如果模拟结果为多头：
  只显示做空信号价（反向信号价）
  做多信号价 = 0

如果模拟结果为空头：
  只显示做多信号价（反向信号价）
  做空信号价 = 0

如果模拟结果为无持仓：
  显示正常的做多和做空信号价
```

### 7.6 移仓示例

```
T日 (2026-03-11):
  主力合约: BU2604 → BU2606
  记录待移仓信息

T+1日 (2026-03-12):
  旧合约BU2604开盘价: 3926
  新合约BU2606开盘价: 3940
  
  操作:
  1. 平BU2604多仓@3926
  2. 模拟BU2606历史交易 → 结果: 空头
  3. 开BU2606空仓@3940
  4. 新合约信号价格只显示做多信号价
```

### 7.7 多合约状态管理
- 每个合约维护独立的阴阳集合状态
- 所有合约状态每日同步更新
- 移仓换月时新合约的阴阳集合状态是完整的

## 8. 时序性要求（核心）

### 8.1 核心原则
**严格遵循时序，禁止使用未来函数**

### 8.2 K线处理时序（详细）

```
输入: KLine_T (第T天的K线数据)

Step 1: 处理临时阴阳集合（如果有）
  - 如果存在上一根K线生成的临时阴阳集合
  - 使用临时集合计算信号价格（反向信号价或开仓信号价）
  - 注意：使用临时集合的状态方向（IsYang），不是当前K线的实际方向

Step 2: 触发判断
  - 使用Step 1计算的信号价格
  - 判断KLine_T是否触发成交
  - 判断逻辑：High >= 做多信号价 或 Low <= 做空信号价

Step 3: 标记临时状态
  - 如果存在临时阴阳集合，标记为已使用（tempUsed = true）
  - 注意：必须在触发判断之后才能标记

Step 4: 更新阴阳集合状态
  - 使用KLine_T的数据更新阴阳集合
  - 判断K线方向（阳线/阴线/十字星）
  - 执行元素合并或降级

Step 5: 成交后处理
  如果成交：
    - 生成交易信号
    - 判断是否生成临时阴阳集合
    - 计算反向信号价（用于T+1日）
  如果不成交：
    - 更新信号价格（用于T+1日）
```

### 8.3 时序关键点

| 时间点 | 操作 | 使用的数据 |
|--------|------|-----------|

### 8.4 禁止事项
- ❌ 禁止在T日使用T+1日的数据
- ❌ 禁止在触发判断前更新阴阳集合
- ❌ 禁止在触发判断前标记临时集合为已使用
- ❌ 禁止在移仓检测日立即执行移仓
- ❌ 禁止使用未来价格计算信号

## 11. 策略优势与风险

### 11.1 优势
- 趋势跟踪能力强，能捕捉大级别行情
- 动态止损机制，风险可控
- 支持多空双向交易，适应各种市场环境
- 信号明确，执行简单

### 11.2 风险
- 震荡市容易频繁止损
- 突发事件可能导致较大回撤
- 移仓换月时可能产生滑点损失
- 杠杆使用不当可能放大亏损

### 11.3 适用场景
- 日线级别的趋势行情
- 流动性好的主流品种
- 中长期投资周期

## 12. 常见问题

### Q1: 为什么要在触发判断后才更新阴阳集合？
A: 这是为了避免使用未来函数。信号价格必须在上一根K线结束后计算完成，用于当前K线的触发判断。如果在触发判断前更新阴阳集合，就相当于使用了当前K线的数据来决定当前K线的交易，这是不合理的。

### Q2: 临时阴阳集合的作用是什么？
A: 临时阴阳集合用于处理逆势开仓的情况。当做多开仓发生在阴线时，说明价格在下跌过程中触发了做多信号，此时下一根K线的信号价格计算需要考虑这种特殊情况。

### Q3: 为什么临时集合有自己的状态方向？
A: 临时集合是把当前K线当成相反方向处理的结果。例如，做多开仓发生在阴线时，临时集合把这根阴线当成阳线处理，所以临时集合的状态方向是"阳线"，而不是当前K线的实际方向"阴线"。信号价格计算时必须使用临时集合的状态方向。

### Q4: 移仓换月为什么要延迟执行？
A: 主力合约的切换是在收盘后才能确定的，如果在当天就执行移仓，就相当于使用了当天的收盘数据来决定当天的交易，这违反了时序原则。

### Q5: 新合约方向为什么要通过模拟交易确定？
A: 每个合约有自己的供需关系和价格走势，新合约的持仓方向应该基于其自身的阴阳集合状态，而不是简单继承旧合约的方向。模拟交易可以确保新合约的方向符合策略逻辑。
